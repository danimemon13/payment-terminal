BrainTree Payment Terminal v1.0

quick & easy credit card payments with BrainTree Payments API and optional PayPal payments

Introduction

Our script allows you to have a quick & easy payment terminal for your clients to pay with major credit cards (Visa, Master Card, American Express, Discover) on your website (without leaving it), as well as paypal payments through paypal gateway. Installation and configuration of the script takes less than 5 minutes (however you do need to have SSL and BrainTree Payments account). There are 3 options to pay: 1) customers can *input price themselves* and type description, 2) you can turn on *pre-defined services* and customers will select service from dropdown and 3) customers can select *recurring services from dropdown*

Optionally PayPal Website Payments Standard can be enabled/disabled for both one time PayPal payments and recurring PayPal payments.

Requirements

1) PHP 5.0+

2) cURL must be enabled to accept payments (contact your hosting provider if unsure).

Fresh Installation

Fresh installation is pretty simple. 

1) Unzip the contents of the file which you downloaded from codecanyon.net.
You will find 2 folders: "braintree-payment-terminal" and "documentation"

2) open braintree-payment-terminal/includes/config.php and edit all needed variables (see explanation below)

3) Upload "braintree-payment-terminal" folder to your FTP server. 

Configuration File Variables

below you will find all editable variables from includes/config.php and their meaning. 

Line 29 - global page meta title
Line 31 - notification email address where you want to receive "new payment received" emails. (can be multiple emails, comma separated)
Line 34 through 39 - non-recurring services array. You can add more by copy/pasting line 33 below it.
Line 43 - if set to true services dropdown will be available on payment page for selection, if false - fillable amount box will appear
Line 46 - Payment type switch, can be ONETIME or RECUR. If set to ONETIME - payment terminal will charge customers single transaction. If set to RECUR - recurring services dropdown will appear instead of amount input box/services dropdown.
Line 60 - Recurring services array. Please note that you have to create recurring plans in braintree account. All created recurring plans will show up in services dropdown by default - if paymnet type is set to "RECUR". If you want to show particular services you need to insert their id's in this array like so: $recur_services = array('id_plan_1_that_I_want_to_display','id_plan_2')
Line 63 - Live/Sandbox indicator. If set to false - you will be using TEST environment, setting this to true will enable LIVE mode.
Line 66 - HTTPS redirection, in case your customers landed on a non-https page. Set this to TRUE if going live.
Line 69 through 72 - Test environment api credentials (sandbox account required. You can get one at https://www.braintreepayments.com/get-started )
Line 76 through 79 - Live environment api credentials (Merchant ID, Private Key, public key, all can be retrieved from account, when logged in)

"PayPal Standard" Settings

In order to enable paypal payments (website payments standard) you need to change Line 75 in config.php to TRUE. Other paypal settings below:

Line 85 - PayPal payment method on/off switch.
Line 86 - PayPal merchant email.
Line 87 - PayPal success url - your customers will be taken to this page after successful paypal payment.
Line 88 - PayPal cancel url - your customers will be taken to this page if they decide not to pay through paypal.
Line 89 - PayPal ipn url - this is IPN listener file which will accept paypal responses about the payments and will notify you about new payments (if you want to update your db somehow - this is the file for paypal).
Line 90 - PayPal custom variable - you can pass any customer related variable here, serves for the purpose of updating database later in ipn listener file (optional).
Line 91 - PayPal currency.
Line 92 - PayPal Live/Sandbox mode switch. If set to true - paypal will run in TEST mode(you have to have paypal developers account and you must be logged in to developer.paypal.com)

Other than the above - you don't need to edit any variables for PayPal.

Connecting payment processing to your database

In order to connect BrainTree payment terminal with your existing website database, to insert new orders or update existing upon successful payment you need to place your "update order" or "insert new order" code into following places:
For regular payments (and non-recuring payments): includes/form.processing.php - between line 188 and line 190
For recurring payments: includes/form.processing.php - between line 419 and 421
For paypal payments standard : paypal_listener.php - between line 61 and 63

Going LIVE (or switching to test mode)

You need to set variable on line 63 in includes/config.php to TRUE to use LIVE MDOE and switch to FALSE to use TEST MODE 

For test purposes you can use 4111111111111111 VISA credit card with any cvv (3 numbers) and any future expiry date. For other test cards please take a look here - https://www.braintreepayments.com/docs/php/reference/sandbox

You will need to create paypal developer account in order to test the paypal in sandbox - please read more here: https://cms.paypal.com/ca/cgi-bin/?cmd=_render-content&content_ID=developer/howto_testing_sandbox
